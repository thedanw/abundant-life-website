# Setup — Agenic Workspace Scaffolding

## When to Use

- Setting up a new project as an agenic workspace from scratch
- Cloning a repo that needs the skill repository linked
- The user asks to "set up this folder as an agenic workspace"
- Re-linking skills after moving the skill repository

## Overview

This skill scaffolds a complete agenic workspace in **any folder** using a low-token approach. It performs three steps:

1. **Link skills** — Create a junction/symlink from `.agents/skills` to the skill repository
2. **Generate `agents.md`** — Create a minimal agent configuration that tells agents how to use the BOSS registry
3. **Update index** — Rebuild `BOSS_INDEX.json` so the new skill is discoverable

The skill repository path is **configurable** — pass it as a parameter so it works across different machines and GitHub clones.

## Workflow

### Step 1: Guardrail Check

Ask: *"Does `.agents/agents.md` already exist?"*
- If **YES** → Workspace is already set up. Ask user if they want to re-link or update.
- If **NO** → Continue.

### Step 2: Collect Parameters

Ask the user (or derive from context) for:

| Parameter | Description | Default |
|-----------|-------------|---------|
| `SKILL_REPO_PATH` | Full path to the skill repository's `.agents/skills` folder | Prompt user |
| `WORKSPACE_ROOT` | Root of the project to set up | Current workspace root |

If the skill repo is a known location (e.g., cloned from GitHub), the user provides the path. The agent **must not hardcode paths** — they vary per machine.

### Step 3: Create `.agents` Directory

```powershell
New-Item -ItemType Directory -Path "$WORKSPACE_ROOT\.agents" -Force
```

### Step 4: Link Skills (Junction/Symlink)

Run the junction script with the provided paths:

```powershell
# PowerShell (Windows)
New-Item -ItemType Junction -Path "$WORKSPACE_ROOT\.agents\skills" -Target "$SKILL_REPO_PATH" -Force
```

```bash
# Linux/macOS
ln -sf "$SKILL_REPO_PATH" "$WORKSPACE_ROOT/.agents/skills"
```

The agent should detect the OS and use the appropriate command.

### Step 5: Generate `agents.md`

Create `.agents/agents.md` with minimal, high-signal content. Follow the template below. **Do not enumerate individual skills** — the BOSS registry handles discovery.

```markdown
# Agentic Workspace Configuration
# Generated by BOSS meta-orchestrator
# Last updated: {{DATE}}

Skills are linked via junction at `.agents/skills/` → `{{SKILL_REPO_PATH}}`

## How to Use

This workspace uses the **BOSS meta-orchestrator** — a low-token skill registry system. Do not enumerate or load skills into context unless needed.

### Workflow

1. **Guardrail check** — Can you solve the task with basic tools (file editing, terminal, conversation)? If yes, solve directly. No skill needed.

2. **Registry lookup** — If the task is complex, query `BOSS_INDEX.json` at `.agents/skills/boss/BOSS_INDEX.json`:
   - Extract 2-5 keywords from the task
   - Match against skill `tags` and `triggers` in the index
   - If a `PROJECT_INDEX.json` exists at `.agents/skills/PROJECT_INDEX.json`, merge it (project skills override on name conflict)

3. **Load only what you need** — For each matched skill:
   - Read only the YAML frontmatter and "When to Use" section of its `SKILL.md`
   - Load the full skill only if step-by-step instructions are required
   - Never load more than 3 skills at once unless the task genuinely requires it

4. **Record combinations** — After completing a significant task, record useful skill combinations in AIngram for cross-session persistence.

### Principles

- **Don't load, register** — Skills are indexed in compact JSON, not dumped into context
- **Match before load** — Keyword-based registry lookup loads only what you'll use
- **Index once, query often** — The registry is a single pre-built file
- **Fail fast on simple tasks** — If basic tools can do it, don't touch the registry
```

### Step 6: Rebuild BOSS Index

Run the update script so the new `setup` skill is registered:

```powershell
pwsh "$WORKSPACE_ROOT\.agents\skills\boss\update-index.ps1"
```

### Step 7: Verify

Confirm the workspace is ready:

```
.agents/
├── agents.md          ← Agent configuration
└── skills/            ← Junction to skill repository
    └── boss/
        ├── BOSS_INDEX.json
        ├── SKILL.md
        ├── setup/     ← This skill (now registered)
        └── ...
```

## Scripts

Helper scripts are available in `meta/setup/scripts/`:

| Script | Purpose |
|--------|---------|
| `New-AgenicWorkspace.ps1` | Full scaffolding: junction + agents.md + index update |
| `New-SkillsJunction.ps1` | Create or repair the skills junction only |

These scripts accept `-WorkspaceRoot` and `-SkillRepoPath` parameters so they work on any machine.
